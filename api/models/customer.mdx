## Overview

The `Clientes` model represents customer entities in the Vantix Sales system. This model is part of the `Vantix.Domain.Entities.Models` namespace and is mapped to the `Clientes` table in the SQL Server database.

## Entity Definition

The complete source code from the Vantix codebase:

```csharp Clientes.cs
using System;
using System.Collections.Generic;

namespace Vantix.Domain.Entities.Models;

public partial class Clientes
{
    public int ClienteId { get; set; }

    public string Nombre { get; set; } = null!;

    public string Apellido { get; set; } = null!;

    public string DocumentoIdentidad { get; set; } = null!;

    public string? Email { get; set; }

    public string? Telefono { get; set; }

    public DateTime? FechaRegistro { get; set; }

    public bool? Activo { get; set; }
}
```

## Database Context Configuration

The complete Entity Framework Core configuration from `VantixDbContext.cs`:

```csharp VantixDbContext.cs
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    modelBuilder.Entity<Clientes>(entity =>
    {
        // Primary Key
        entity.HasKey(e => e.ClienteId).HasName("PK__Clientes__71ABD0872790C1F1");

        // Unique Constraint on DocumentoIdentidad
        entity.HasIndex(e => e.DocumentoIdentidad, "UQ__Clientes__049E81A9FF79380A").IsUnique();

        // Default Values
        entity.Property(e => e.Activo).HasDefaultValue(true);
        entity.Property(e => e.FechaRegistro)
            .HasDefaultValueSql("(getdate())")
            .HasColumnType("datetime");
        
        // String Length Constraints
        entity.Property(e => e.Nombre).HasMaxLength(100);
        entity.Property(e => e.Apellido).HasMaxLength(100);
        entity.Property(e => e.DocumentoIdentidad).HasMaxLength(20);
        entity.Property(e => e.Email).HasMaxLength(150);
        entity.Property(e => e.Telefono).HasMaxLength(20);
    });

    OnModelCreatingPartial(modelBuilder);
}
```

## Validation Summary

  <Accordion title="Required Fields">
    The following fields are required when creating a customer:
    
    - `nombre` - First name
    - `apellido` - Last name
    - `documentoIdentidad` - Identity document (must be unique)
    
    All other fields are optional or auto-generated.
  </Accordion>

  <Accordion title="Maximum Lengths">
    | Field | Max Length |
    |-------|------------|
    | nombre | 100 |
    | apellido | 100 |
    | documentoIdentidad | 20 |
    | email | 150 |
    | telefono | 20 |
  </Accordion>

  <Accordion title="Unique Constraints">
    - **DocumentoIdentidad**: Must be unique across all customer records
    - Violation results in HTTP 409 Conflict error
  </Accordion>

  <Accordion title="Default Values">
    - **activo**: Defaults to `true`
    - **fechaRegistro**: Defaults to current database timestamp
    - These fields are automatically set if not provided
  </Accordion>

  <Accordion title="Nullable Fields">
    The following fields can be `null`:
    
    - `email`
    - `telefono`
    - `fechaRegistro` (though it has a default value)
    - `activo` (though it has a default value)
  </Accordion>
</AccordionGroup>

## Database Table Schema

```sql
CREATE TABLE Clientes (
    ClienteId INT IDENTITY(1,1) PRIMARY KEY,
    Nombre NVARCHAR(100) NOT NULL,
    Apellido NVARCHAR(100) NOT NULL,
    DocumentoIdentidad NVARCHAR(20) NOT NULL UNIQUE,
    Email NVARCHAR(150) NULL,
    Telefono NVARCHAR(20) NULL,
    FechaRegistro DATETIME DEFAULT GETDATE(),
    Activo BIT DEFAULT 1
);
```
