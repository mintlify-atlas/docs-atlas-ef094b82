## Overview

Vantix uses a **database-first** approach with Entity Framework Core scaffolding. The `scaffold.bat` script automates the generation of entity models and DbContext from an existing SQL Server database.

## The Scaffold Script

From `scaffold.bat`:

```batch
@echo off
echo 🚀 Generando modelos en Domain e Infraestructura...

dotnet ef dbcontext scaffold "Server=GARIEN-DESKTOP;Database=VantixV1;Trusted_Connection=True;TrustServerCertificate=true;" Microsoft.EntityFrameworkCore.SqlServer ^
  --project Vantix.Infrastructure ^
  --output-dir ../Vantix.Domain/Entities/Models ^
  --context-dir Persistence ^
  --context VantixDbContext ^
  --force ^
  --no-pluralize ^
  --namespace Vantix.Domain.Entities.Models ^
  --context-namespace Vantix.Infrastructure.Persistence

echo ✅ ¡Revisa tus carpetas ahora!
pause
```

## Scaffold Parameters Explained

| Parameter | Value | Description |
|-----------|-------|-------------|
| Connection String | `Server=GARIEN-DESKTOP;...` | SQL Server database to scaffold from |
| Provider | `Microsoft.EntityFrameworkCore.SqlServer` | Database provider for SQL Server |
| `--project` | `Vantix.Infrastructure` | Project where DbContext is generated |
| `--output-dir` | `../Vantix.Domain/Entities/Models` | Directory for entity classes |
| `--context-dir` | `Persistence` | Directory for DbContext (relative to project) |
| `--context` | `VantixDbContext` | Name of the generated DbContext class |
| `--force` | | Overwrite existing files without prompting |
| `--no-pluralize` | | Keep table names as-is (don't pluralize) |
| `--namespace` | `Vantix.Domain.Entities.Models` | Namespace for entity classes |
| `--context-namespace` | `Vantix.Infrastructure.Persistence` | Namespace for DbContext |

<Info>
The `--force` flag will overwrite existing files. Be careful if you've made manual changes to generated models.
</Info>

## Database-First Workflow

<Steps>
  <Step title="Design Database Schema">
    Create or modify your database schema in SQL Server:

    ```sql
    CREATE TABLE Clientes (
        ClienteId INT PRIMARY KEY IDENTITY(1,1),
        Nombre NVARCHAR(100) NOT NULL,
        Apellido NVARCHAR(100) NOT NULL,
        DocumentoIdentidad NVARCHAR(20) UNIQUE NOT NULL,
        Email NVARCHAR(150),
        Telefono NVARCHAR(20),
        Activo BIT DEFAULT 1,
        FechaRegistro DATETIME DEFAULT GETDATE()
    );
    ```
  </Step>

  <Step title="Update Connection String (Optional)">
    If your database server or name is different, edit `scaffold.bat`:

    ```batch
    dotnet ef dbcontext scaffold "Server=YOUR_SERVER;Database=YOUR_DB;..." ...
    ```

    <Tip>
    Consider using a configuration file instead of hardcoding the connection string. See the Database Configuration guide.
    </Tip>
  </Step>

  <Step title="Run Scaffold Script">
    Execute the batch file from your project root:

    ```bash
    # From the solution root directory
    .\scaffold.bat
    ```

    You'll see output like:
    ```
    🚀 Generando modelos en Domain e Infraestructura...
    Build started...
    Build succeeded.
    ✅ ¡Revisa tus carpetas ahora!
    ```
  </Step>

  <Step title="Review Generated Files">
    Check the generated files:

    - **Entities**: `Vantix.Domain/Entities/Models/Clientes.cs`
    - **DbContext**: `Vantix.Infrastructure/Persistence/VantixDbContext.cs`
  </Step>

  <Step title="Remove Hardcoded Connection String">
    <Warning>
    The scaffolding process generates a hardcoded connection string in `VantixDbContext.cs`. Remove it and use configuration instead.
    </Warning>

    Delete or comment out the `OnConfiguring` method:

    ```csharp
    // protected override void OnConfiguring(DbContextOptionsBuilder optionsBuilder)
    //     => optionsBuilder.UseSqlServer("...");
    ```
  </Step>
</Steps>

## Customizing the Scaffold Process

### Scaffold Specific Tables

To generate models for specific tables only:

```batch
dotnet ef dbcontext scaffold "Server=..." Microsoft.EntityFrameworkCore.SqlServer ^
  --project Vantix.Infrastructure ^
  --output-dir ../Vantix.Domain/Entities/Models ^
  --context-dir Persistence ^
  --context VantixDbContext ^
  --table Clientes ^
  --table Ventas ^
  --table Productos
```

### Use Configuration File for Connection String

Create a more secure scaffold script that reads from configuration:

```batch
@echo off
echo 🚀 Generando modelos desde configuración...

dotnet ef dbcontext scaffold "Name=ConnectionStrings:ventixDb" Microsoft.EntityFrameworkCore.SqlServer ^
  --project Vantix.Infrastructure ^
  --startup-project Vantix.Api ^
  --output-dir ../Vantix.Domain/Entities/Models ^
  --context-dir Persistence ^
  --context VantixDbContext ^
  --force ^
  --no-pluralize ^
  --namespace Vantix.Domain.Entities.Models ^
  --context-namespace Vantix.Infrastructure.Persistence

echo ✅ ¡Modelos generados!
pause
```

<Tip>
Using `Name=ConnectionStrings:ventixDb` reads the connection string from `appsettings.json`, avoiding hardcoded credentials.
</Tip>

### Preserve Manual Changes

Use partial classes to extend generated models without losing changes:

```csharp
// Clientes.Custom.cs - Your custom code that won't be overwritten
namespace Vantix.Domain.Entities.Models;

public partial class Clientes
{
    public string NombreCompleto => $"{Nombre} {Apellido}";
    
    public bool EsActivo() => Activo ?? false;
}
```

## Troubleshooting

### Build Failed Error

```bash
Build FAILED.
```

**Solution**: Ensure your project compiles before scaffolding:
```bash
dotnet build Vantix.Infrastructure
```

### Connection Failed

```bash
A network-related or instance-specific error occurred...
```

**Solution**: 
1. Verify SQL Server is running
2. Check connection string server name
3. Ensure Windows Authentication is enabled or provide credentials

### No Tables Found

```bash
No tables were found in the database.
```

**Solution**:
1. Verify you're connecting to the correct database
2. Check that tables exist in the database
3. Ensure the database user has read permissions

### Namespace Conflicts

```bash
The type or namespace name already exists...
```

**Solution**: Use `--force` to overwrite existing files, or delete old generated files first.

## Advanced: Linux/macOS Script

Create `scaffold.sh` for cross-platform support:

```bash
#!/bin/bash
echo "🚀 Generating models in Domain and Infrastructure..."

dotnet ef dbcontext scaffold "Server=localhost;Database=VantixV1;User Id=sa;Password=YourPassword;TrustServerCertificate=true" Microsoft.EntityFrameworkCore.SqlServer \
  --project Vantix.Infrastructure \
  --output-dir ../Vantix.Domain/Entities/Models \
  --context-dir Persistence \
  --context VantixDbContext \
  --force \
  --no-pluralize \
  --namespace Vantix.Domain.Entities.Models \
  --context-namespace Vantix.Infrastructure.Persistence

echo "✅ Check your folders now!"
```

Make it executable:
```bash
chmod +x scaffold.sh
./scaffold.sh
```
