<Note>
  This quickstart guide will help you set up and run the Vantix Sales API locally in under 5 minutes.
</Note>

## Prerequisites

Before you begin, ensure you have the following installed:

- **.NET 10 SDK** - [Download here](https://dotnet.microsoft.com/download/dotnet/10.0)
- **SQL Server** - [Download SQL Server Express](https://www.microsoft.com/sql-server/sql-server-downloads)
- **Git** - [Download here](https://git-scm.com/downloads)

## Quick Setup

<Steps>
  <Step title="Clone the Repository">
    Clone the Vantix Sales API repository to your local machine:

    ```bash
    git clone <your-repository-url>
    cd SistemasVentasV2Angular
    ```
  </Step>

  <Step title="Configure Database Connection">
    Update the connection string in `Vantix.Api/appsettings.Development.json` to match your SQL Server instance:

    ```json Vantix.Api/appsettings.Development.json
    {
      "Logging": {
        "LogLevel": {
          "Default": "Information",
          "Microsoft.AspNetCore": "Warning"
        }
      },
      "ConnectionStrings": {
        "ventixDb": "Server=YOUR_SERVER_NAME;Database=VantixV1;Trusted_Connection=True;TrustServerCertificate=True"
      },
      "Jwt": {
        "Key": "Esta_Es_Una_Llave_Super_Secreta_De_Al_Menos_32_Caracteres_12345!"
      }
    }
    ```

    <Warning>
      Replace `YOUR_SERVER_NAME` with your actual SQL Server instance name (e.g., `localhost`, `DESKTOP-ABC123`, or `.\\SQLEXPRESS`).
    </Warning>
  </Step>

  <Step title="Create the Database">
    Create the VantixV1 database in SQL Server. You can use SQL Server Management Studio (SSMS) or run this SQL command:

    ```sql
    CREATE DATABASE VantixV1;
    ```

    Then create the Clientes table:

    ```sql
    USE VantixV1;
    GO

    CREATE TABLE Clientes (
        ClienteId INT PRIMARY KEY IDENTITY(1,1),
        Nombre NVARCHAR(100) NOT NULL,
        Apellido NVARCHAR(100) NOT NULL,
        DocumentoIdentidad NVARCHAR(20) NOT NULL UNIQUE,
        Email NVARCHAR(150),
        Telefono NVARCHAR(20),
        FechaRegistro DATETIME DEFAULT GETDATE(),
        Activo BIT DEFAULT 1
    );
    ```
  </Step>

  <Step title="Restore Dependencies">
    Navigate to the API project directory and restore all NuGet packages:

    ```bash
    cd Vantix.Api
    dotnet restore
    ```
  </Step>

  <Step title="Run the API">
    Start the API server:

    ```bash
    dotnet run
    ```

    You should see output similar to:

    ```
    info: Microsoft.Hosting.Lifetime[14]
          Now listening on: https://localhost:7160
    info: Microsoft.Hosting.Lifetime[14]
          Now listening on: http://localhost:5198
    info: Microsoft.Hosting.Lifetime[0]
          Application started. Press Ctrl+C to shut down.
    ```
  </Step>

  <Step title="Access the OpenAPI Documentation">
    Open your browser and navigate to the OpenAPI endpoint:

    ```
    https://localhost:7160/openapi/v1.json
    ```

    <Tip>
      The OpenAPI endpoint is only available in Development mode. You can also use tools like Swagger UI or Postman to visualize and test the API endpoints.
    </Tip>
  </Step>
</Steps>

## Understanding the Application Architecture

The Vantix Sales API follows a clean architecture pattern with the following projects:

<CardGroup cols={2}>
  <Card title="Vantix.Api" icon="server" color="#2563eb">

  </Card>
  
  <Card title="Vantix.Domain" icon="cube" color="#3b82f6">

## Key Configuration Files

### Program.cs

The main entry point configures services and the HTTP pipeline:

```csharp Vantix.Api/Program.cs
var builder = WebApplication.CreateBuilder(args);

// Add services to the container.
builder.Services.AddControllers();
// Learn more about configuring OpenAPI at https://aka.ms/aspnet/openapi
builder.Services.AddOpenApi();

var app = builder.Build();

// Configure the HTTP request pipeline.
if (app.Environment.IsDevelopment())
{
    app.MapOpenApi();
}

app.UseHttpsRedirection();
app.UseAuthorization();
app.MapControllers();

app.Run();
```

### Project Configuration

The API targets .NET 10 with nullable reference types enabled:

```xml Vantix.Api/Vantix.Api.csproj
<Project Sdk="Microsoft.NET.Sdk.Web">
  <PropertyGroup>
    <TargetFramework>net10.0</TargetFramework>
    <Nullable>enable</Nullable>
    <ImplicitUsings>enable</ImplicitUsings>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Microsoft.AspNetCore.OpenApi" Version="10.0.3" />
  </ItemGroup>
</Project>
```

## Getting Help

If you encounter any issues during setup:

- Check that SQL Server is running and accessible
- Verify your connection string is correct
- Ensure .NET 10 SDK is properly installed by running `dotnet --version`
- Review the console output for any error messages

<Info>
  The application runs on HTTPS (port 7160) and HTTP (port 5198) by default. You can modify these ports in `Vantix.Api/Properties/launchSettings.json`.
</Info>
